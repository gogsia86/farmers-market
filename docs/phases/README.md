## ğŸ“ Documentation Standards

### File Naming

```yaml
Phase Files:
  Format: PHASE_X_DESCRIPTION.md
  Example: PHASE_7_MVP_LAUNCH_PLAN.md

Session Summaries:
  Format: SESSION_X_SUMMARY.md or DAY_X_SESSION.md
  Example: SESSION_2_SUMMARY.md

Execution Documents:
  Format: PHASE_X_COMPONENT_ACTION.md
  Example: PHASE_7_INFRASTRUCTURE_EXECUTION.md
```

### Document Lifecycle

```yaml
1. Active Phase (Root Directory):
  - PHASE_X_PROGRESS_TRACKER.md (primary)
  - PHASE_X_[COMPONENT]_EXECUTION.md (active guides)
  - PHASE_X_NEXT_SESSION_CHECKLIST.md (session planning)

2. Reference Phase (docs/phases/phase-X/):
  - Main planning documents
  - Executive summaries
  - Quick reference guides
  - Complete documentation packages

3. Historical Phase (docs/phases/phase-X/):
  - Moved from root when phase completes
  - session-summaries/ subdirectory
  - Archived for reference

4. Archive (docs/archive/YYYY-MM/):
  - After 6+ months or when superseded
  - Historical value only
```
